Basic Scene

To create an actual visual scene, we need to place an object into the world and if we want to have an interactive dynamic scene we need to know, how to register and process inputs.

We are using ExampleMinimumGraphicsSetup (which also uses ExampleBaseScene) as a reference to show you how it can be done:

...\(CrossForge root folder)\Examples\exampleMinimumGraphicsSetup.hpp

Loading Assets

Loading assets requires the use of two classes: T3DMesh and SAssetIO. T3DMesh is used to store a triangle mesh and with that various properties. SAssetIO handles import and export of meshes and image files.

We will also assign the mesh to a StaticActor which is used in the scene graph that we will look at next.

//StaticActor m_Skydome;
//StaticActor m_Cube;

// load skydome and a textured cube
T3DMesh<float> M;

SAssetIO::load("Assets/ExampleScenes/SimpleSkydome.glb", &M);                   //SAssetIO::load(assetpath, *object to save asset)
setMeshShader(&M, 0.8f, 0.04f);                                                 //ExampleSceneBase::setMeshShader(*mesh, Roughness, Metallic)     
//function of example base scene which sets material properties and assigns shader, optional
M.computePerVertexNormals();
m_Skydome.init(&M);                                                             //assign/save Mesh to StaticActor
M.clear();                                                                      //clear storage to reuse

SAssetIO::load("Assets/ExampleScenes/TexturedUnitCube.gltf", &M);
setMeshShader(&M, 0.1f, 0.04f);
M.computePerVertexNormals();
m_Cube.init(&M);
M.clear();

This example uses a white skydome as a skybox by loading it as a mesh. To add an actual Skybox, you can check out the concept here.

Scene Graph

In Crossforge the scene is stored as a scene graph consisting of the base class SceneGraph and the node interface ISceneGraphNode which is represented by the node types SGNGeometry and SGNTransformation.

//SceneGraph m_SG; (declared in ExampleBaseScene)

//SGNTransformation m_RootSGN;
//SGNGeometry m_SkydomeSGN;
//SGNGeometry m_CubeSGN;
//SGNTransformation m_CubeTransformSGN;

// build scene graph
m_RootSGN.init(nullptr);                                                        
m_SG.init(&m_RootSGN);                                                          //initialize the scene graph with the root
//could add transformations for the whole scene

// add skydome
m_SkydomeSGN.init(&m_RootSGN, &m_Skydome);                                      
//SGNGeometry::init can receive multiple arguments, it will always need the parent node (or nullptr if it is the root)
//and in this case we assign the StaticActor to the geometry, rest will be set to default
m_SkydomeSGN.scale(Vector3f(50.0f, 50.0f, 50.0f));                              //and scale it

// add cube
m_CubeTransformSGN.init(&m_RootSGN, Vector3f(0.0f, 3.0f, 0.0f));                //transform node
//here we pass only 1 Vector in addition to the parent node to SGNTransformation::init, which sets the translation value
m_CubeSGN.init(&m_CubeTransformSGN, &m_Cube);
//similar to skydome, but we assign the transform node as parent so transformation applies
m_CubeSGN.scale(Vector3f(3.0f, 3.0f, 3.0f));

We can also animate the scene with the scene graph by setting transformation values as delta:

// rotate about the y-axis at 45 degree every second
Quaternionf R;                                                                  
//quaternion as  representation of orientations and rotations
R = AngleAxisf(GraphicsUtility::degToRad(45.0f / 60.0f), Vector3f::UnitY());    //get quaternion for rotation around Y axis
m_CubeTransformSGN.rotationDelta(R);                                            
//set rotationDelta which will apply rotation regularly through SceneGraph::Update()

Input

The classes Keyboard and Mouse represent as the name indicates your basic input devices. They are part of the window and have all the standard keys and actions listed, while constantly checking their state which is received from the system automatically. Through the functions Keyboard::keyPressed(Key) and Mouse::buttonState(Button) you can check if the Key/Button is pressed.

First we write a function for camera movement:

void defaultCameraUpdate(VirtualCamera* pCamera, Keyboard* pKeyboard, Mouse* pMouse, const float MovementSpeed = 0.4f, const float RotationSpeed = 1.0f, const float SpeedScale = 4.0f) {
			if (nullptr == pCamera) throw NullpointerExcept("pCamera");
			if (nullptr == pKeyboard) throw NullpointerExcept("pKeyboard");
			if (nullptr == pMouse) throw NullpointerExcept("pMouse");

			float S = 1.0f;
			if (pKeyboard->keyPressed(Keyboard::KEY_LEFT_SHIFT)) S = SpeedScale;
            //pressing left shift will apply a scale for faster movement

			if (pKeyboard->keyPressed(Keyboard::KEY_W)) pCamera->forward(S * MovementSpeed);
			if (pKeyboard->keyPressed(Keyboard::KEY_S)) pCamera->forward(S * -MovementSpeed);
			if (pKeyboard->keyPressed(Keyboard::KEY_A)) pCamera->right(-S * MovementSpeed);
			if (pKeyboard->keyPressed(Keyboard::KEY_D)) pCamera->right(S * MovementSpeed);
            //WASD keys for camera movement, using VirtualCamera's integrated movement functions

			if (pMouse->buttonState(Mouse::BTN_RIGHT)) {                           //when right mouse button is pressed(hold)
				const Eigen::Vector2f MouseDelta = pMouse->movement();             //track mouse movement
				pCamera->rotY(GraphicsUtility::degToRad(-0.1f * RotationSpeed * MouseDelta.x()));
				pCamera->pitch(GraphicsUtility::degToRad(-0.1f * RotationSpeed * MouseDelta.y()));
                //tilt camera according to mouse movement
				pMouse->movement(Eigen::Vector2f::Zero());                         //reset movement so further movement is not added in next cycle
			}
		}//defaultCameraUpdate

Next we want to set some functional inputs:

void defaultKeyboardUpdate(Keyboard* pKeyboard) {
			if (nullptr == pKeyboard) throw NullpointerExcept("pKeyboard");
			if (pKeyboard->keyPressed(Keyboard::KEY_F9, true)) {
				m_RenderWin.vsync(!m_RenderWin.vsync());                           //toggle VSync when pressing F9
			}
			if (m_RenderWin.keyboard()->keyPressed(Keyboard::KEY_F10, true)) {
				m_RenderDev.activePass(RenderDevice::RENDERPASS_FORWARD);
				std::string ScreenshotURI = "Screenshot_" + std::to_string(m_ScreenshotCount++) + ".webp";
				takeScreenshot(ScreenshotURI);                                     //take screenshot when pressing F10
			}


			if (pKeyboard->keyPressed(Keyboard::KEY_ESCAPE)) {                     //close Window on Esc
				m_RenderWin.closeWindow();                                         //m_RenderWin.shutdown()=true
			}
		}//defaultKeyboardUpdate

To see our changes we have to add some update functions to our main loop:

void run(void) {
			while (!m_RenderWin.shutdown()) {
				m_RenderWin.update();
				m_SG.update(60.0f / m_FPS);                                         //update the scene graph(with 60 fps)

				defaultCameraUpdate(&m_Cam, m_RenderWin.keyboard(), m_RenderWin.mouse());
                //camera update function

				m_RenderDev.activePass(RenderDevice::RENDERPASS_SHADOW, &m_Sun);
				m_SG.render(&m_RenderDev);

				m_RenderDev.activePass(RenderDevice::RENDERPASS_GEOMETRY);
				m_SG.render(&m_RenderDev);

				m_RenderDev.activePass(RenderDevice::RENDERPASS_LIGHTING);
                //render the scene graph through the different render passes

				m_RenderWin.swapBuffers();

				//updateFPS();

				defaultKeyboardUpdate(m_RenderWin.keyboard());                      //keyboard update function

			}//while[main loop]
		}//run
